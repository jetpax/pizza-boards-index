# PiZZA Boards — Arduino Boards Manager index

Install [PiZZA](https://github.com/jetpax/PiZZa) (Arduino-on-Zephyr for the
Raspberry Pi Zero 2 W) directly from the Arduino IDE.

## Install

1. Arduino IDE → **Settings / Preferences** → **Additional boards manager URLs**,
   paste:

   ```
   https://raw.githubusercontent.com/jetpax/pizza-boards-index/main/package_pizza_index.json
   ```

2. **Tools → Board → Boards Manager**, search **PiZZA**, click **Install**.
   This pulls the core and the `aarch64-zephyr-elf` toolchain (from the
   official [Zephyr SDK](https://github.com/zephyrproject-rtos/sdk-ng)).

3. **Tools → Board → PiZZA (Raspberry Pi Zero 2 W)**.

You also need a Pi Zero 2 W running the **PiZZA OS first-run image** on its
microSD card — see the [PiZZa releases](https://github.com/jetpax/PiZZa/releases).
The loader firmware on the SD runs your sketch as a runtime-loaded `.llext`;
no kernel re-flash per sketch.

## What's in the manifest

- **platform** `pizza:zephyr` — the PiZZA core, served as a release asset on
  [jetpax/ArduinoCore-zephyr](https://github.com/jetpax/ArduinoCore-zephyr).
- **tool** `aarch64-zephyr-elf` 0.17.0 — the AArch64 cross toolchain, pointing
  straight at Zephyr SDK release tarballs (mac x86_64/arm64, linux x86_64/aarch64,
  windows x86_64). Not re-hosted here.

The manifest is regenerated by the release workflow in ArduinoCore-zephyr on
each `pizza-v*` tag (version, URL, checksum, size).

## Serial & upload — read this first

This board does **not** present a USB device, so it works differently from a
typical Arduino:

- **Serial is the UART, not USB.** Arduino `Serial` is the mini-UART on
  **GPIO14 (TX) / GPIO15 (RX) @ 115200**. Connect a USB-serial adapter to those
  pins and open it at 115200 (the IDE Serial Monitor won't auto-detect it —
  point it at that port manually). There is no USB-CDC Serial yet.
- **Upload writes to the SD card.** The IDE **Upload** button copies the
  compiled sketch to the SD card mounted on your computer (macOS
  `/Volumes/RECOVERY`, Linux `/media/<user>/RECOVERY`) as `sketch.llext`, then
  ejects it. Workflow: SD in your computer → Upload → move SD to the Pi → power
  on. (Direct USB upload is planned for a later release.)
- **LED_BUILTIN** is the on-board green ACT LED (BCM GPIO 29).

Start from **File → Examples → PiZZA → Blink / HelloSerial**.

## Library support (v0.1.1)

Wired and working on the rpi_zero_2w variant: **SPI**, **Wire** (I²C),
**Serial** (UART), **GPIO**, **RTC**, and **WiFi** (brcmfmac).

Libraries with no backing driver on this board are not shipped (Camera,
Storage, SDRAM, Ethernet, CAN, LED Matrix). There is no `analogRead`/ADC
backing.
